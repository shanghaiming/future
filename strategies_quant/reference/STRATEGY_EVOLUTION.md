## Phase 1：信号时代 (43-50%)

### V37 — 基线共识策略：+43.1%
- **方法**: 9策略共识入场（≥3策略同意），简单退出
- **指标**: WR=63%, avgW=8.3%, 192 trades, 16d持仓
- **关键**: 多策略投票共识入场

### V53 — 智能退出系统：+49.8%
- **方法**: V37共识入场 + 基于极值理论(EVT)的概率退出
- **分段止损**: 前3天宽止损，3-7天正常，7天后紧止损+跟踪
- **共识翻转退出**: 所有策略翻转到卖出时立即退出
- **指标**: WR=64%, 195 trades
- **意义**: 纯信号框架的天花板，无ML

### V60 — 截面因子组合：+23.9%
- **方法**: 3因子百分位排名（MOM20, VDP_Delta, KER）— Fisher正交族
- **结论**: 排名改善了选股，但没有智能退出仅+23.9%。排名和择时是独立问题

## Phase 2：ML 引入 (53-115%)

### V61b — 首次XGBoost分类：+53.0%
- **方法**: XGBoost二分类预测涨跌，59个策略信号作为特征
- **指标**: WR=50%, avgW=17.9%（大赢利补偿低胜率）
- **意义**: ML首次用于选股

### V64 — LightGBM LambdaRank 突破：+115.4%
- **方法**: 从XGBoost分类切换到 **LightGBM LambdaRank**（学习排序）
- **指标**: WR从50%跳到 **69%** — 单次最大胜率提升
- **关键洞察**: 排序模型 >> 分类模型，用于选股

## Phase 3：ML 优化 (276-418%)

### V67 — 超参优化：+276%
- **方法**: 优化LightGBM超参数
- **指标**: WR=71%

### V69 — 5模型集成：+308%
- **方法**: 多模型集成+保守配置
- **指标**: WR=66%

### V70 — 超保守模型：+307%
- **方法**: 极简模型 num_leaves=7, feature_fraction=0.4, min_data_in_leaf=100
- **特征**: 138个（16手动 + 59买信号 + 59卖信号 + 2市场 + 2共识）
- **训练**: Walk-forward, 每42天重训, 3年窗口
- **指标**: WR=72%, Edge=5.24%, MaxDD=30%
- **结论**: 更简单的模型泛化更好

### V71 — Bug修复版LightGBM：+335.7%
- **修复**: 策略去重（59→43唯一）, 训练泄漏修复（train_end = train_di - FWD_DAYS - 1）
- **新增**: 内置bug检查（随机基线+年度分解）
- **意义**: 首个真正清洁的ML版本
- **关键**: 去除重复策略+修复泄漏 = +28pp

### V74 — ATR自适应止损：+360.7%
- **方法**: V71基础 + ATR自适应止损
- **对比**: V73尝试Alpha101特征 — 失败（+329.8%，增加噪声）
- **指标**: Edge从4.63%(V71)提升到5.58%(V74), MaxDD从41%改善到21%

### V75 — CatBoost集成：+418.5%
- **方法**: LightGBM + CatBoost集成 + 有序提升
- **指标**: WR=76%, Edge=5.15%, TPY=37, DD=16%
- **关键**: 正交模型平均提升鲁棒性，重训LGB模型后两者都改善

## Phase 4：Meta标注时代 (394-583%, 含泄漏)

### V76 — 多种子修剪：+394.2%
- **方法**: 特征修剪 + 多种子集成
- **意义**: Meta过滤的首次尝试

### V77 — Meta标注引入：+463.8%
- **方法**: 二级模型过滤主模型的选股（meta-labeling）
- **指标**: Edge跳到 **7.06%**, WR=81%
- **BUG**: meta模型在70%数据上训练，在100%数据上预测（train-on-apply泄漏）

### V79 — Meta + ATR：+500.9%（首次突破500%）
- **方法**: Meta标注 + ATR自适应止损
- **指标**: WR=81%, Edge=6.68%, TPY=32
- **BUG**: 仍有V77的train-on-apply泄漏

### V80 — XGB排序：+244.2% ❌ 失败
- **结论**: XGBoost排序不如LightGBM LambdaRank

### V81 — 快速重训：+441.0% ❌ 不如V79

- **结论**: 更频繁重训反而损害Edge（6.68%→5.88%）

### V82 — 加权评分：+419.2% ❌ 失败
- **方法**: 连续加权代替二元meta过滤
- **结论**: 更差

### V83 — Walk-Forward Meta：+331.8% ❌ 失败
- **方法**: Walk-forward meta模型，~500样本/窗口
- **结论**: 样本太少，meta反而有害

### V84 — Top-K选择：+497.1%
- **方法**: 接近V79，但DD大幅改善（10% vs 38%）

### V85 — Meta退出：+501.6%（无改善）
- **方法**: Meta指导退出时机
- **结论**: 过早止盈，Edge从6.03%降到5.84%

### V86 — Top-5 Meta训练：+583.0%（峰值，但泄漏）
- **核心创新**: 在top-5选股上训练meta模型（9589 vs 1913样本）
- **指标**: WR=84%, Edge=6.51%, TPY=34, DD=16%
- **年度**: 所有年份盈利（2018:+224%, 2019:+591%, 2020:+1166%, 2021:+1805%）
- **BUG**: 与V77相同的train-on-apply泄漏，膨胀了~115pp

### V87 — Top-5 Meta + Top-K：+514.7%
- **结论**: 即使有top-5 meta训练，top-1选择仍优于top-K

## Phase 5：清洁Meta时代 (343-491%)

### V88 — 无泄漏Walk-Forward Meta：+467.6%
- **修复**: 扩展窗口walk-forward, 固定META_ROUNDS=100, 无早停
- **无meta基线**: +413.5%（meta真正贡献+54pp）
- **最佳配置**: wf_meta0.55_atr, WR=77%, Edge=6.02%, TPY=33, DD=11%
- **泄漏影响**: V86的+583%被膨胀了~115pp，真实值~467.6%

### V89 — 3-LGB配置集成：+490.6%（最佳清洁结果）✨
- **方法**: 训练3个不同超参的LGB排序模型，平均排名
  - Config A: num_leaves=7, feature_fraction=0.5
  - Config B: num_leaves=15, feature_fraction=0.3
  - Config C: num_leaves=31, feature_fraction=0.7
- **指标**: Edge=**7.51%**, WR=83%, TPY=27, DD=11%
- **关键洞察**: **配置多样性 >> 算法多样性**

### V90 — 回归Meta：+448.0%
- **方法**: V88主模型 + meta用回归预测收益幅度
- **结论**: 不如二元meta

### V91 — 多算法（XGB+CB+LGB）：+430.0% ❌ 不如V89
- **方法**: 5模型集成（3xLGB + XGBRanker + CatBoostRanker）
- **结论**: 算法多样性增加的噪声 > 信号
- **教训**: **配置多样性(V89) > 算法多样性(V91)**

### V92 — 级联选择：+486.7%
- **方法**: top-1被meta过滤后，级联到top-2, top-3...top-K

### V93 — 集成+级联（固定种子）：+404.0%

## 关键发现

### 1. 真实清洁上限是 ~490%（V89）
V102的+533%是幸运种子，V86的+583%被泄漏膨胀了~115pp。

### 2. Edge/TPY 权衡是核心约束
| 指标 | 最高值 | 版本 | 代价 |
|------|-------|------|------|
| Edge | 7.58% | V98 | TPY仅26 |
| TPY | 36 | V92 | Edge仅5.73% |
| 800%需求 | ≥7.5% + ≥30 | — | 需同时满足 |

### 3. 有效技术排名
| 技术 | 贡献 | 证据 |
|------|------|------|
| LambdaRank排序 | 最大 | V64 vs V61b: +53%→+115% |
| Bug修复+去重 | 巨大 | V71 vs V70: +307%→+336% |
| ATR自适应止损 | 大 | V74 vs V71: Edge +0.95pp |
| Meta标注（无泄漏） | 大 | V88 vs 无meta: +54pp |
| 配置多样性 | 大 | V89: Edge=7.51% |
| 多周期训练 | 中 | V98: Edge=7.58% |
| CatBoost集成 | 中 | V75: +418% |
| 算法多样性 | 无/负 | V91 < V89 |
| 级联选择 | 混合 | 增加TPY但降Edge |
| 二元分类 | 负 | V96 << V89 |

### 4. 失败尝试清单
- XGBoost排序(V80), 快速重训(V81), 加权评分(V82)
- Walk-forward小窗口meta(V83), Meta退出时机(V85)
- Alpha101特征(V73), 算法多样性(V91)
- 二元分类(V96), 开盘-开盘标签(V108)
- 更多模型(9个不如3个): V100 < V89

### 5. 种子随机性是最大不可控因素
V98/V102/V103架构相同，但从+442%到+533%不等。

## 冲刺800%的可行路径

基于历史记录，要同时达到 Edge≥7.5% AND TPY≥30：

1. **保留V89的3-config集成**（Edge=7.51%基础）
2. **加回V98的多周期**（Edge提升到7.58%）
3. **用V88的无泄漏meta**过滤（增加WR/Edge）
4. **找到V102的幸运种子**或系统搜索最优种子
5. **适度级联**（V92思路，但控制不超过top-3，避免Edge下降太多）
6. **收盘-收盘标签**（V108证明开盘-开盘更差）

核心矛盾：更多信号选择(TPY↑) vs 更高质量(Edge↑) — 需要找到两者同时提升的"甜点"。
